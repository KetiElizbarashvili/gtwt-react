To Clarify

What We Need to Decide

These are blockers or open product decisions before we can finalize the build.

1

Real Backend / Database

All data is hardcoded. No real users, no persistence. Do we need a DB before launch? Which service?

2

Real Authentication

Any email/password works right now. Magic links? Self-registration? Who controls the user list?

3

Real-time Messaging

Messages disappear on refresh. Do we need live WebSocket sync, or is async "post + refresh" OK?

4

Who Are the Users?

Speakers only? All attendees? Expected simultaneous users? Invite-only or public access?

5

Admin / Announcements

Who can post official announcements? Is there an admin panel, or a special organizer account?

Nice to Have Later
Speaker bio/photo data source (CMS? spreadsheet?)
Conference schedule / agenda screen
Push notifications
Connect / follow feature (beyond DM)
Next Steps

Recommended Path Forward

1
Decide on backend
Supabase = simplest option (auth + DB + realtime in one). Firebase or custom API are alternatives.
2
Confirm user scope & login method
Who gets access? Magic link invite, self-registration, or manual list?
3
Get real speaker data
A spreadsheet with names, bios, photos, LinkedIn is enough to start. We'll wire it in.
4
Decide: real-time vs async chat
Real-time (Supabase channels) is harder but impressive. Async is simpler. Your call.
5
Design sign-off
Confirm stage colors, logo usage, and any missing brand elements.
🚀
The MVP shell is ready.
All screens are built and deployed. The next milestone is wiring real data and a real backend — everything else is already in place.
